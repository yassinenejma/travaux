Travail à faire N°1 DevOps - Docker

Commandes Docker utilisées pendant le TP

Travail à faire N°1

1

Lancement du conteneur Nginx

J'ai utilisé la commande docker run -d -p 8081:80 nginx pour lancer un conteneur Nginx en mode détaché (-d) et mapper le port 8081 de l'hôte vers le port 80 du conteneur.

2

Vérification des conteneurs en cours d'exécution

J'ai utilisé la commande docker ps pour lister tous les conteneurs en cours d'exécution, voir leurs ID, noms, ports mappés et autres informations.

3

Lancement d'un second conteneur Nginx

J'ai utilisé la commande docker run -d -p 8082:80 nginx pour lancer un second conteneur Nginx en mappant cette fois le port 8082 de l'hôte vers le port 80 du conteneur.

Travail à faire N°2

1

Lancement du conteneur MySQL en arrière-plan

J'ai utilisé la commande docker run -d --name my-mysql -e MYSQL_ROOT_PASSWORD=mypassword -p 8082:3306 mysql pour lancer un conteneur MySQL en mode détaché avec un mot de passe racine défini et un mappage du port 8082.

2

Lancement interactif du conteneur MySQL

J'ai utilisé la commande docker run -it --name my-mysql -e MYSQL_ROOT_PASSWORD=mypassword mysql pour lancer un conteneur MySQL en mode interactif avec un terminal attaché.

3

Démarrage du conteneur MySQL en mode interactif

J'ai utilisé la commande docker start -ai my-mysql pour démarrer un conteneur MySQL existant en mode interactif et connecté à son terminal.

4

Utilisation de MySQL dans le conteneur

Après avoir lancé le conteneur MySQL, j'ai pu utiliser le shell MySQL pour exécuter des commandes SQL et interagir avec la base de données.
